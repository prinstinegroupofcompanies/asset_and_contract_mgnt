# Asset & Contract Management System (ACMS)

A web-based system for managing assets, stock, vehicles, fuel, and contracts with audit trails and role-based access control.

## Tech Stack

| Layer | Technologies |
|-------|-------------|
| Frontend | React 18, React Router, Axios, React Query |
| Backend | Node.js, Express, JWT, SQLite / PostgreSQL |
| Hosting | **Vercel** (frontend) + **Render** (backend + database) |

## Local Development

### Prerequisites

- Node.js 18+
- npm

### Setup

```bash
# Install dependencies
npm run install-all

# Copy environment template
cp env.template server/.env

# Start both servers
npm run dev
```

- Frontend: http://localhost:3000
- Backend API: http://localhost:5000/api/health

### Default Login

| Username | Password |
|----------|----------|
| `admin` | `admin123` |

Change the default password before going to production.

### Step 1 вҖ” Deploy Backend on Render

1. Push this repository to GitHub.
2. Go to [Render Dashboard](https://dashboard.render.com) вҶ’ **New** вҶ’ **Blueprint**.
3. Connect the repo: `prinstinegroupofcompanies/asset_and_contract_mgnt`.
4. Render reads `render.yaml` and creates:
   - **Web service** `acms-api` (Node.js API)
   - **PostgreSQL database** `acms-database`
5. Click **Apply** and wait for the first deploy to finish.
6. Copy your API URL, e.g. `https://acms-api.onrender.com`.
7. In the **acms-api** service вҶ’ **Environment**, set:

   | Variable | Value |
   |----------|-------|
   | `CLIENT_URL` | Your Vercel URL (set after Step 2), e.g. `https://your-app.vercel.app` |

   For preview deployments, use comma-separated URLs:
   ```
   https://your-app.vercel.app,https://your-app-git-main.vercel.app
   ```

8. Verify the API is running:
   ```bash
   curl https://acms-api.onrender.com/api/health
   ```

The database is initialized automatically on first startup.

### Step 2 вҖ” Deploy Frontend on Vercel

1. Go to [Vercel Dashboard](https://vercel.com) вҶ’ **Add New** вҶ’ **Project**.
2. Import the same GitHub repository.
3. Vercel reads `vercel.json` at the repo root. Confirm:
   - **Build Command:** `cd client && npm install && npm run build`
   - **Output Directory:** `client/build`
4. Add this environment variable (required at build time):

   | Variable | Value |
   |----------|-------|
   | `REACT_APP_API_URL` | `https://acms-api.onrender.com/api` |

   Replace with your actual Render API URL.

5. Click **Deploy**.
6. Copy your Vercel URL, e.g. `https://asset-and-contract-mgnt.vercel.app`.

### Step 3 вҖ” Link Frontend and Backend

1. Return to **Render** вҶ’ **acms-api** вҶ’ **Environment**.
2. Update `CLIENT_URL` to your Vercel URL:
   ```
   CLIENT_URL=https://your-app.vercel.app
   ```
3. Save changes (Render will redeploy automatically).
4. Open your Vercel URL and log in with the default credentials.
5. **Change the admin password immediately.**

- **Backend:** Push to `main` вҖ” Render auto-deploys (if enabled).
- **Frontend:** Push to `main` вҖ” Vercel auto-deploys.

If you change `REACT_APP_API_URL`, trigger a new Vercel deployment (env vars are baked in at build time).

## Troubleshooting

### CORS errors in the browser

- Ensure `CLIENT_URL` on Render exactly matches your Vercel URL (no trailing slash).
- Include preview URLs as comma-separated values if needed.

### `ERR_CONNECTION_REFUSED` or API unreachable

- Confirm the Render service is running (free tier sleeps after inactivity; first request may take ~30s).
- Verify `REACT_APP_API_URL` ends with `/api`.
- Check `https://your-api.onrender.com/api/health`.

### Login returns 401 / invalid credentials

- Database may not be seeded yet вҖ” check Render logs for initialization errors.
- Default credentials: `admin` / `admin123`.

### 429 Too Many Requests

- Rate limiting is enabled. Wait a few minutes or retry from a different network.
